# 최종 시연 시나리오

## 시나리오 1. 정상 센서 데이터 수집

1. Sensor Simulator가 정상 유해가스 데이터를 생성한다.
2. Collector API가 데이터를 수신한다.
3. Raw 테이블에 원천 데이터를 저장한다.
4. Staging 검증을 통과한다.
5. Mart 테이블에 현재 센서 상태가 생성된다.
6. 대시보드에 정상 상태가 표시된다.

## 시나리오 2. CO 위험 증가

1. Sensor Simulator가 CO 상승 데이터를 생성한다.
2. CO 값이 25ppm 이상이면 warning으로 판단한다.
3. CO 값이 200ppm 이상이면 danger로 판단한다.
4. Risk Event를 생성한다.
5. AlarmRecord에 알람을 저장한다.
6. 대시보드에 위험 알람을 표시한다.

## 시나리오 3. 작업자 위험구역 진입

1. 작업자 위치 데이터가 생성된다.
2. 작업자가 지오펜스 내부로 이동한다.
3. 해당 구역의 센서값이 위험 상태인지 확인한다.
4. 위치 조건과 센서 조건이 동시에 만족되면 알람을 생성한다.
5. 지도에서 작업자와 위험구역을 강조한다.

## 시나리오 4. 전력 이상 탐지

1. 특정 채널의 전력 사용량이 증가한다.
2. 최근 평균보다 높은 전력 사용량을 감지한다.
3. AI 또는 통계 기반 이상탐지 결과를 생성한다.
4. AIPredictionResult에 저장한다.
5. 대시보드에 전력 이상 징후를 표시한다.

## 시나리오 5. 알람 확인과 피드백

1. 운영자가 알람을 확인한다.
2. 알람 상태가 acknowledged로 변경된다.
3. 운영자가 조치 완료를 기록한다.
4. true_alarm 또는 false_alarm 피드백을 저장한다.
5. 피드백은 AI 학습 데이터셋 후보로 활용된다.

## 시나리오 6. RAG 대응 매뉴얼 검색

1. CO 위험 알람이 발생한다.
2. 알람 상세 화면에서 CO 대응 매뉴얼을 검색한다.
3. 대응 절차와 문서 출처를 표시한다.

## 시나리오 7. 데이터 품질 확인

1. 중복 event_id 데이터가 들어온다.
2. Staging 검증 단계에서 중복으로 분류한다.
3. 지연 timestamp 데이터가 들어온다.
4. 결측값 데이터가 들어온다.
5. 오류 데이터는 dead_letter_events에 저장된다.
6. 품질 리포트에서 중복/지연/오류 건수를 확인한다.
